# Персонализированные настройки — мастер-план (Bambusito228)

Документ для поэтапной реализации расширенных настроек обменника.  
Цель: **много настраиваемого**, но **удобно** — не одна простыня, а логичные разделы + быстрые действия на обзоре.

---

## Решения заказчика (зафиксировано)

| Вопрос | Ответ |
|--------|--------|
| Блоки настроек | Все: брендинг, обмен, курсы, кошельки, ордера, чат, уведомления, тексты сайта, техрежим, безопасность |
| Структура админки | **Гибрид**: частые настройки на «Обзоре», остальное — в подразделах вкладки «Настройки» |
| Где редактировать | **Веб `/admin`** (полный доступ) + **Telegram Mini App** (быстрые поля) |
| Наценка / лимиты | **Одна глобальная** наценка + общие min/max суммы |
| Кошельки | **Один адрес** на всё (как сейчас) |
| Telegram-бот командами | Не расширять — только веб + Mini App |
| Приоритет v1 | Решает исполнитель (см. этапы ниже) |

## Текущее состояние (baseline)

**Ключи в `settings` (SQLite):**

| Ключ | Назначение |
|------|------------|
| `markup_percent` | Глобальная наценка % |
| `usd_rub_rate` | Курс USD/RUB |
| `site_name` | Название сайта |
| `deposit_wallet` | Единый адрес оплаты |
| `order_ttl_minutes` | TTL ордера |
| `chat_operator_name` | Имя в шапке чата |
| `rate_provider` | auto / bybit / binance / okx / coingecko |
| `admin_username`, `admin_password_hash` | Вход в веб-админку |

**Проблема UX:** всё в одной вкладке «Настройки» — длинный скролл, смешаны курс, кошелёк, пароль.

**Файлы для правок:**

- `server/db.js` — defaults + миграции ключей
- `server/routes/admin.js` — `GET/PATCH /api/admin/settings`, валидация
- `server/routes/api.js` — публичный конфиг для фронта (только безопасные поля)
- `admin/index.html`, `admin/admin.js`, `admin/admin.css` — веб-админка
- `admin/tg/index.html`, `admin/tg/app.js`, `admin/tg/app.css` — Mini App
- `index.html`, `assets/js/exchange-app.js`, `assets/js/chat-widget.js` — применение на сайте
- `server/telegram.js` — уважать `notif_*` при отправке в Telegram

## Информационная архитектура (гибрид)

### Боковое меню (без изменений)

`Обзор` · `Ордера` · `Чат` · `Настройки`

### Обзор — «быстрые карточки» (редактирование inline или модалка)

Показывать текущее значение + кнопка «Изменить»:

1. **Наценка** — `markup_percent`
2. **USD/RUB** — `usd_rub_rate`
3. **Источник курса** — `rate_provider` + статус probe (как сейчас)
4. **Техрежим** — `maintenance_mode` (toggle)
5. **Кошелёк** — последние 8 символов `deposit_wallet`

Не дублировать всю форму — только то, что меняют чаще всего.

### Вкладка «Настройки» — горизонтальные подвкладки (pills)

```
[ Обмен ] [ Курсы ] [ Ордера ] [ Чат ] [ Уведомления ] [ Сайт ] [ Безопасность ]
```

Каждая подвкладка — 1–3 `panel settings-section`, не больше одного экрана без скролла на десктопе.

## Каталог настроек по разделам

### 1. Обмен (`exchange`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `markup_percent` | number | `1.5` | Наценка от рынка, % |
| `exchange_min_usd` | number | `50` | Мин. сумма сделки в USD-эквиваленте |
| `exchange_max_usd` | number | `50000` | Макс. сумма |
| `enabled_directions` | json | `["*"]` | Опционально v2: список отключённых пар; v1 — не делать |

**Сайт:** валидация суммы в `exchange-app.js` до создания ордера.  
**API:** отдавать min/max в публичном `/api/config`.

### 2. Курсы (`rates`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `rate_provider` | enum | `auto` | Источник курса |
| `usd_rub_rate` | number | `92.5` | Ручной USD/RUB |
| `usd_rub_auto` | bool | `false` | v2: тянуть с ЦБ/API; v1 — только ручной |
| `rate_refresh_sec` | number | `60` | Интервал кэша курсов на сервере (30–300) |

Панель probe провайдеров — **оставить как есть**, перенести в подвкладку «Курсы».

### 3. Кошельки (`wallets`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `deposit_wallet` | string | `''` | Единый адрес (QR на странице ордера) |

Подсказка в UI: «Один адрес для всех монет». Без multi-wallet.

### 4. Ордера (`orders`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `order_ttl_minutes` | number | `30` | Таймер на `order.html` |
| `order_status_labels` | json | `{}` | v2: кастомные подписи статусов; v1 — не делать |

### 5. Чат (`chat`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `chat_operator_name` | string | `Bambusito228 Support` | Имя в виджете |
| `chat_welcome_message` | text | `Здравствуйте! Чем можем помочь?` | Первое сообщение при открытии |
| `chat_offline_message` | text | `Оператор ответит в ближайшее время.` | Когда вне часов работы |
| `chat_work_hours` | string | `09:00-21:00` | Часы работы (локальное время сервера или `Europe/Moscow` — зафиксировать в коде) |
| `chat_show_online` | bool | `true` | Показывать зелёную точку «онлайн» |

**Сайт:** `chat-widget.js` читает из `/api/config` → `chat`.

### 6. Уведомления (`notifications`)

Флаги `notif_*` = `1` / `0`. При `0` — не слать в Telegram.

| Ключ | По умолчанию | Событие |
|------|--------------|---------|
| `notif_new_order` | `1` | Новый ордер |
| `notif_order_status` | `0` | Смена статуса (v2) |
| `notif_chat_message` | `1` | Сообщение в чате |
| `notif_bybit_deposit` | `1` | Депозит на Bybit |
| `notif_maintenance` | `1` | Включение техрежима (v2) |

Правки в `server/telegram.js`, `server/bybitDeposits.js`, `server/routes/chat.js` — обёртка `if (getSetting('notif_…') === '1')`.

### 7. Сайт / тексты (`site`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `site_name` | string | `Bambusito228` | Title, шапка |
| `site_tagline` | string | `Быстрый обмен криптовалют` | Подзаголовок на главной |
| `accent_color` | string | `#22c55e` | CSS variable `--accent` |
| `contact_telegram` | string | `''` | @username или ссылка |
| `contact_email` | string | `''` | Email в футере |
| `rules_text` | text | `''` | Правила обмена (markdown-lite или plain) |
| `faq_text` | text | `''` | FAQ блок (v2: отдельные вопросы; v1 — один textarea) |

**Сайт:** `index.html` — динамическая подстановка через `exchange-app.js` или отдельный `site-config.js`.

### 8. Техрежим (`maintenance`)

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `maintenance_mode` | bool | `0` | Блокировать создание ордеров |
| `maintenance_message` | text | `Обмен временно приостановлен. Попробуйте позже.` | Баннер на главной |

**API:** `POST /api/orders` → 503 + message если maintenance.  
**Админка и Mini App** — доступны всегда.

### 9. Безопасность (`security`)

| Ключ | Описание |
|------|----------|
| `admin_username` | Логин (readonly в UI или смена v2) |
| Смена пароля | Отдельная форма, **только веб** (не Mini App) |

2FA, IP whitelist — **не в v1**, пункт «Скоро» в UI.

## Публичный API (`GET /api/config`)

Расширить ответ (без секретов):

```json
{
  "site_name": "...",
  "site_tagline": "...",
  "accent_color": "#22c55e",
  "markup_percent": 1.5,
  "exchange_min_usd": 50,
  "exchange_max_usd": 50000,
  "maintenance_mode": false,
  "maintenance_message": "...",
  "chat": {
    "operatorName": "...",
    "welcomeMessage": "...",
    "offlineMessage": "...",
    "workHours": "09:00-21:00",
    "showOnline": true
  },
  "contacts": {
    "telegram": "",
    "email": ""
  },
  "rules_text": "",
  "faq_text": ""
}
```

Админский `GET /api/admin/dashboard` — полный объект `settings` + группировка по секциям (опционально).

`PATCH /api/admin/settings` — принимать плоский объект или `{ section, values }`; валидация по секции.

## UX-правила (обязательны для исполнителя)

1. **Не больше 5–7 полей** на одном подэкране.
2. **Подсказка `muted`** — что увидит клиент на сайте.
3. **Групповое сохранение** — одна primary-кнопка внизу секции.
4. **Опасные действия** — техрежим: confirm «Выключить обмен для клиентов?»
5. **Цвет** — input type=color + текстовый hex рядом.
6. **Длинные тексты** — textarea 4–6 строк, не WYSIWYG в v1.
7. **Mini App** — крупные touch-targets, без пароля.
